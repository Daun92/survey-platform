# Survey Platform - UI/UX 가이드

> 엔터프라이즈급 설문조사 플랫폼의 컴포넌트, 레이아웃, CX 설계 가이드
> 기술 스택: Next.js 15 + Tailwind CSS v4 + shadcn/ui
> 최종 업데이트: 2026-03-14

## 1. 디자인 철학 & 레퍼런스

### 1.1 벤치마크 플랫폼

| 플랫폼 | 핵심 UX 패턴 | 채택 포인트 |
|--------|-------------|------------|
| **Typeform** | 1문항 1페이지 대화형, 브랜드 커스텀, 완료율 34% 향상 | 응답 페이지(`/s/[token]`)에 대화형 옵션 제공 |
| **SurveyMonkey** | 전통 폼, AI 질문 추천, 엔터프라이즈 분석 대시보드 | 관리자 대시보드 데이터 밀도, AI 추천 패턴 |
| **Tally** | 미니멀·즉시생성, Notion 스타일 블록 에디터 | 에디터의 블록 기반 UX, 제로 러닝커브 |

### 1.2 SaaS 대시보드 레퍼런스

| 제품 | 참고 패턴 |
|------|----------|
| **Linear** | Calm Design — 최소 크롬, 작업에 집중, 커맨드 팔레트(Cmd+K) |
| **Vercel** | Strategic Minimalism — 필요한 데이터만, 노이즈 제로 |
| **Notion** | AI를 인프라로 — 인라인 제안, 별도 모드 없음 |
| **HubSpot** | 역할 기반 UI — 같은 제품, 역할별 완전히 다른 기본 뷰 |

### 1.3 적용할 2026 SaaS UI 트렌드 (7가지)

1. **Calm Design** — 인지 부하 최소화, 작업 필수 정보만 기본 표시
2. **AI as Infrastructure** — AI 기능을 별도 배지 없이 자연스럽게 내장
3. **Command Palette (Cmd+K)** — 모든 액션을 키보드로 접근, 퍼지 검색
4. **역할 기반 적응형 UI** — Admin/Manager/User별 다른 기본 뷰
5. **Progressive Disclosure** — 빈 상태가 가르치고, 고급 옵션은 호버에
6. **Emotional Design** — 완료 애니메이션, 대화형 카피, 브랜드 일러스트
7. **Strategic Minimalism** — 화면당 하나의 명확한 다음 액션

## 2. 컴포넌트 체계

### 2.1 shadcn/ui 컴포넌트 설치 계획

#### P0 — 즉시 필요 (Phase 2~3)

| 컴포넌트 | 용도 |
|---------|------|
| `button` | 모든 CTA, 상태별 변형(default/destructive/outline/ghost) |
| `card` | 통계 카드, 질문 카드, 설문 카드 |
| `input` + `textarea` | 모든 폼 입력 |
| `label` | 폼 레이블 |
| `select` | 드롭다운 선택 |
| `dialog` + `alert-dialog` | 확인/경고 모달 |
| `dropdown-menu` | 컨텍스트 메뉴 (질문 카드 ··· 버튼) |
| `badge` | 상태 표시 (Draft/Active/Closed) |
| `switch` | 필수 여부 토글 |
| `table` | 응답 목록, 사용자 관리 |
| `toast` / `sonner` | 알림 토스트 (저장 완료, 에러 등) |
| `skeleton` | 로딩 스켈레톤 |
| `tabs` | 설문 상세 탭 (편집/배포/리포트) |
| `progress` | 응답 진행률, 업로드 진행 |

#### P1 — 점진적 도입 (Phase 3~5)

| 컴포넌트 | 용도 |
|---------|------|
| `separator` | 섹션 구분선 |
| `scroll-area` | 에디터 스크롤 영역 |
| `tooltip` | 아이콘 버튼 힌트 |
| `avatar` | 사용자 프로필 |
| `command` | Cmd+K 커맨드 팔레트 |
| `popover` | 날짜 선택기, 필터 |
| `sheet` | 모바일 사이드바, 상세 패널 |
| `breadcrumb` | 페이지 계층 탐색 |
| `chart` (recharts) | 리포트 차트 (Phase 4) |

### 3.2 레이아웃 변형

| 레이아웃 | 적용 경로 | 특징 |
|---------|----------|------|

**CX 포인트:**
- StatCard에 7일 스파크라인 미니차트 포함
- 증감률은 초록(+)/빨강(-) 컬러 코딩
- 빈 대시보드 → EmptyState "첫 설문을 만들어보세요" + CTA
- 자동 새로고침 (30초 간격 또는 WebSocket)

**CX 포인트:**

- 일괄 작업: 체크박스 선택 → 상단에 "N개 선택됨 [삭제] [보관]"
- 빈 상태별 다른 메시지: "아직 설문이 없습니다" vs "검색 결과 없음"

```
│ │ N NPS         │ │                         │ │                 │   │
│ │               │ │                         │ │                 │   │
│ │ -- 기타 --    │ │                         │ │                 │   │
```

**CX 포인트:**
- 자동 저장 (2초 디바운스) + 상단에 "저장됨" / "저장 중..." 표시
- 미리보기 패널을 모바일(380px) 프레임으로 표시 — 응답자 시점 정확 재현
- 질문 카드 접기/펼치기 — 펼친 카드만 편집, 나머지는 제목+타입 요약
- 드래그 핸들(=) + 우상단 컨텍스트 메뉴(...): 복제, 삭제, 위/아래 이동
- 타입 팔레트에서 드래그 또는 클릭으로 추가 — 추가 시 자동 스크롤+포커스
- 에디터 풀스크린: 사이드바 자동 접힘, FAB 숨김


## 6. 인터랙션 패턴

### 6.1 토스트 알림

- **위치**: 우하단 (bottom-right)
- **자동 닫힘**: 5초
- **종류**: success(초록), error(빨강), info(파랑), warning(노랑)
- **Undo 지원**: 삭제 작업 시 "되돌리기" 버튼 포함
- **라이브러리**: sonner (shadcn/ui 추천)

### 6.2 확인 다이얼로그

| 액션 | 메시지 | 버튼 스타일 |
|------|--------|-----------|
| 삭제 | "정말 삭제하시겠습니까?" | Destructive (빨강) |
| 발행 | "설문을 발행하시겠습니까? 발행 후에는 질문을 수정할 수 없습니다." | Accent (오렌지) |
| 마감 | "설문을 마감하시겠습니까?" | Warning 아이콘 + Primary |

### 6.3 로딩 상태

| 상황 | 패턴 |
|------|------|
| 초기 로딩 | Skeleton (페이지 구조에 맞춘 형태) |
| 버튼 로딩 | Spinner + 텍스트 변경 ("저장 중...") |
| 테이블 로딩 | 행 3개 Skeleton |
| 페이지 전환 | 상단 Progress bar (NProgress 스타일) |

### 6.4 빈 상태 (Empty State)

- lucide 대형 아이콘 + 제목 + 설명 + CTA 버튼
- 컨텍스트별 다른 메시지:
  - 설문 없음: "첫 번째 설문을 만들어보세요"
  - 검색 결과 없음: "검색 조건에 맞는 결과가 없습니다"
  - 응답 없음: "아직 응답이 수집되지 않았습니다"

### 6.5 커맨드 팔레트 (Cmd+K)

- shadcn/ui `command` (cmdk 기반)
- **섹션**: 페이지 이동, 설문 검색, 최근 항목, 빠른 작업
- 퍼지 검색, 키보드 네비게이션
- 최근 3개 항목 자동 표시
