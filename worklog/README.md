# Worklog - 개발 히스토리 로그

## 목적
프로젝트의 모든 개발 과정을 기록하여 히스토리와 맥락을 관리합니다.

## 파일 명명 규칙
```
YYYY-MM-DD_phaseX_작업명.md
```
예: `2026-03-14_phase0_planning.md`

## 항목 형식

```markdown
# YYYY-MM-DD Phase X - 작업 제목

## 목표
- 이번 세션의 목표

## 수행 내역
### 1. [작업명]
- **명령어**: `실행한 명령어`
- **결과**: 성공/실패 및 주요 출력
- **파일 변경**: 생성/수정/삭제된 파일 목록

## 의사결정 기록
| 결정사항 | 선택지 | 선택 이유 |

## 이슈 & 해결
- [이슈 내용] → [해결 방법]

## 다음 단계
- [ ] 다음에 할 작업

## Git 커밋
- `해시` type: 커밋 메시지
```

## 커밋 규칙
- 워크로그 파일은 `docs:` 접두사로 커밋
- 예: `docs: Phase 0 기획/설계 워크로그 작성`
