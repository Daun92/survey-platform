# Worklog - 개발 히스토리 로그

## 목적
프로젝트의 모든 개발 과정을 기록하여 히스토리와 맥락을 관리합니다.

## 파일 명명 규칙

### 단일 에이전트 작업
```
YYYY-MM-DD_phaseX_작업명.md
```
예: `2026-03-14_phase0_planning.md`

### 멀티 에이전트 동시 작업
```
YYYY-MM-DD_phaseX_역할_작업명.md
```
예:
- `2026-03-15_phase1_frontend_auth-ui.md`
- `2026-03-15_phase1_backend_auth-api.md`
- `2026-03-15_phase1_shared_types.md`

> 멀티 에이전트 작업 시 각 에이전트가 별도 파일을 사용하여 Git 충돌을 방지합니다.

## 항목 형식

```markdown
# YYYY-MM-DD Phase X - 작업 제목

## 목표
- 이번 세션의 목표

## 수행 내역
### 1. [작업명]
- **명령어**: `실행한 명령어`
- **결과**: 성공/실패 및 주요 출력
- **파일 변경**: 생성/수정/삭제된 파일 목록

## 의사결정 기록
| 결정사항 | 제시된 선택지 | 사용자 선택 | 선택 이유/맥락 |

## 이슈 & 해결
- [이슈 내용] → [해결 방법]

## 다음 단계
- [ ] 다음에 할 작업

## Git 커밋
- `해시` type: 커밋 메시지
```

## 워크로그 갱신 시점
1. **세션 시작 시**: 이전 워크로그의 "다음 단계"를 읽고 현재 세션 파일 생성
2. **기능 구현 완료 시**: 구현한 기능, 변경 파일, 테스트 결과 기록
3. **사용자 의사결정 발생 시**: Claude가 선택지를 제시하고 사용자가 선택한 경우 즉시 기록
4. **세션 종료 시**: 다음 단계, 미완료 항목 정리

## 커밋 규칙
- 워크로그 파일은 `docs:` 접두사로 커밋
- 예: `docs: Phase 0 기획/설계 워크로그 작성`

## 멀티 에이전트 워크로그 추가 항목

멀티 에이전트 작업 시 워크로그에 다음 섹션을 추가합니다:

```markdown
## 에이전트 정보
- **에이전트 ID**: agent-{역할}-{번호}
- **담당 영역**: apps/frontend/ (또는 해당 영역)

## 다른 에이전트 의존성
- [에이전트 ID]: [의존 내용] — [상태: 대기중/완료]

## 다음 에이전트에게
- [ ] 이어서 할 작업 설명
- 주의사항: ...
```
